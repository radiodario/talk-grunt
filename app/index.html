The Trio

They're all based on node.js

Yeoman

  • Templating and bootstraping
  • Series of generators
  • Bootstraps everything, from your build system to your .gitIgnore

Yeoman Generators

  • Wizard-ish generators for your projects
  • A whole bunch of them!
  • Distributed as npm packages
  • They start with the prefix `generator` 
    npm seach generator
  • When you find one that you like, install it 
    npm install generator-webapp

Running a generator

  • Once installed, run it
  • Some generators are better than others.
  • Run one: 
    yo webapp
  • Once run, do: 
    npm install && bower install

Bower

  • Depencency Manager
  • Built by the guys at Twitter
  • Able to keep track with any git repo that uses semver

Grunt

  • Task Runner
  • Serve, Test, Minify, Watch, Reload
  • CI-Friendly
  • Loads of plugins.

Gruntfile.js

  • Configured via Gruntfile.js
  • Think of it like a makefile
  • Nothing stops you from building a makefile for your js project.

Grunt Tasks

  • This is a js file.
  • You have task-named properties.
  • And you can add your own.
  • When a task is run, Grunt looks for its configuration under a property of the same name.

Running tasks and targets

  • Call a task like this: 
    grunt concat
    and it'll call all targets for this task, in sequence.
  • Call a task with a target: 
    grunt concat:foo
    and it'll run that specific target.
